# Wiki Schema

## Identity
- **Path:** /exp/mu2e/app/users/oksuzian/muse_050125/prodtools/wiki
- **Domain:** Mu2e prodtools operational knowledge — campaigns, incidents, decisions, production runs
- **Source types:** meeting notes, docdb PDFs, Slack threads, email snippets, POMS incident notes, campaign postmortems, decision records
- **Created:** 2026-04-21

## Relationship to other stores
- **Code** → `utils/`, `bin/` (source of truth, not duplicated here)
- **Usage reference** → `EXAMPLES.md` (regenerated by `/refresh-examples`)
- **Behavioral preferences and short reference facts** → `~/.claude/projects/*/memory/` (auto-loaded every session)
- **This wiki** → durable operational/tribal knowledge that doesn't fit the above: decisions with rationale, incident root causes, campaign-specific context, collaboration notes.

## Page Frontmatter
Every wiki page must start with:
```
---
title: <page title>
tags: [tag1, tag2]
sources: [source-slug1]
updated: YYYY-MM-DD
---
```

## Cross-References
Use `[[slug]]` where slug = filename without `.md`.
Example: `[[run1bai-campaign]]` → `wiki/pages/run1bai-campaign.md`

## Log Entry Format
```
## [YYYY-MM-DD] <operation> | <title>
```
Operations: init, ingest, query, update, lint

Quick view of recent entries:
```
grep "^## \[" wiki/log.md | tail -10
```

## Index Categories
- **Campaigns** — per-campaign notes (MDC2020xx, MDC2025xx, Run1Bxx): status, configuration quirks, rationale behind knob choices
- **Incidents** — production issues, POMS failures, data-handling problems with root cause and resolution
- **Decisions** — ADR-style architecture/workflow choices with context, alternatives considered, chosen path, rationale
- **Runs** — notable production runs with scale, outcome, lessons
- **Sources** — summary pages for ingested external documents (meeting notes, docdb links, Slack exports)
- **Analyses** — filed query answers that compound knowledge

## Conventions
- `raw/` is immutable — skills never modify it
- `log.md` is append-only — never rewritten, only appended
- `index.md` is updated on every operation that adds or changes pages
- All pages live flat in `pages/` — no subdirectories
- `overview.md` reflects the current synthesis across all sources
- Skills find this file by looking at `<repo-root>/wiki/SCHEMA.md`. If missing, tell the user to run `/wiki-init`.
